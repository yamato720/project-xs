## 2. 周期模拟器

已完成文件：

- `src/include/base/CycleSimulator.h`
- `src/base/CycleSimulator.cpp`

当前语义：

- 维护全局周期号 `current_cycle`
- 支持最大运行周期 `max_cycles`
- 支持 `-1` 语义表示无限运行
- 逐拍推动所有已注册 `Kernel`
- 拍末统一调用各 kernel 的 `end_cycle()`
- 支持 `reset()`
- 支持 `initialize_zero()`
- 支持 `terminate()`

其中：

- `reset()` 负责把 kernel 状态恢复到初始态
- `initialize_zero()` 负责把寄存器型端口初始化为“0 可见”

## 3. Kernel / KernelComponent 抽象

已完成文件：

- `src/include/base/Kernel.h`
- `src/base/Kernel.cpp`
- `src/include/base/KernelComponent.h`
- `src/base/KernelComponent.cpp`

### 3.1 Kernel

`Kernel` 代表一个可被周期模拟器逐拍推动的上层模块。

当前支持：

- 单拍执行入口 `run_single()`
- 可挂接多个 `KernelComponent`
- 自带一个默认 `PortGroup`
- 可选单独绑定一个 input port / output port
- 支持 `reset()` / `initialize_zero()` / `end_cycle()`

### 3.2 KernelComponent

`KernelComponent` 代表 kernel 内部某个较小的周期单元。

当前支持：

- `latency`
- `first_delay_align`
- 相位推进 `phase()`
- `phase_valid()`
- 自带一个默认 `PortGroup`
- 可选单独绑定一个 input port / output port
- 支持 `emit_output()`
- 支持 `reset()` / `initialize_zero()` / `end_cycle()`

## 4. 基础端口抽象

已完成文件：

- `src/include/base/Port.h`
- `src/base/Port.cpp`

### 4.1 设计收敛

端口层从一开始的通用 `ScalarPort` 收敛为更明确的两种时序语义：

- `WirePort`
- `RegPort`

当前保留了一个共同基类 `Port`，但实际可见的时序模型只有这两种。

### 4.2 当前能力

每个端口当前具备：

- 固定方向 `Input / Output`
- 固定数据类型
- 固定位宽
- 绑定到某个实际变量
- 私有可见存储区
- 私有待提交存储区
- 严格的类型/位宽匹配检查

### 4.3 时序语义

#### WirePort

- 输出端本拍 `emit_bound_value()` 后立刻可见
- 输入端本拍 `sync_input()` 后即可看到上游当前值

#### RegPort

- 输出端本拍 `emit_bound_value()` 后不会立刻可见
- 要等 `end_cycle()` 之后，下一拍才会变为可见值
- 输入端本拍 `sync_input()` 看到的是上一拍已经提交的值

### 4.4 初始化语义

新增能力：

- `initialize_zero()`

行为：

- `WirePort`：等价于清空
- `RegPort`：初始化为当前拍即可见的 `0`

如果没有做初始化，测试层可把不可见值显示为 `z`

## 5. 无协议端口组 PortGroup

已完成文件：

- `src/include/base/PortGroup.h`
- `src/base/PortGroup.cpp`

### 5.1 设计目标

`PortGroup` 只承载“普通端口”，不直接承载协议含义。

适合：

- 组件的多输入/多输出
- kernel 的一组普通内部接口
- 测试中对一组信号的统一管理

### 5.2 当前能力

- 可命名
- 支持多输入、多输出
- 支持：
  - `add_input()`
  - `add_output()`
  - `sync_inputs()`
  - `end_cycle()`
  - `clear()`
  - `initialize_zero()`
- 支持按索引访问：
  - `input_at(i)`
  - `output_at(i)`
- 支持按名字访问：
  - `find_input(name)`
  - `find_output(name)`
  - `get_input(name)`
  - `get_output(name)`

## 7. ABC 测试

已完成文件：

- `test/abctest/main.cpp`

### 7.1 当前测试结构

当前测试包含：

- `KernelAComponent`
- `KernelBComponent`
- `KernelCComponent`
- `AbcDemoKernel`

### 7.2 组件行为

#### A

- `latency = 1`
- 每拍输出当前周期号
- 通过自己的默认 `PortGroup` 提供两路输出：
  - 一路给 B
  - 一路给 C

#### B

- `latency = 2`
- 在 `phase 0` 锁存 A 的输入
- 在 `phase 1` 输出 `锁存值 + 4`

#### C

- `latency = 3`
- `first_delay_align = 1`
- 在 `phase 0` 锁存 A 的输入
- 在 `phase 1` 输出 `锁存值 * 3`

### 7.3 wire/reg 对照

当前 `AbcDemoKernel` 内部维护两组 3 输出的结果端口组：

- 一组 `wire`
- 一组 `reg`

作用：

- 直接观察“同拍可见”和“下一拍可见”的差别

### 7.4 初始化后的表现

在 `main` 中当前显式执行：

1. `simulator.reset()`
2. `simulator.initialize_zero()`

因此：

- `cycle 0` 时 `reg` 组显示为 `0`
- 如果以后不调用 `initialize_zero()`，则测试层可以把无效值显示为 `z`
